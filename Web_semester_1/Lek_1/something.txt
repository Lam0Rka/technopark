HTTP - протокол передачи текстовых документов

Чекнуть MIME - типы html, JS, xml - это всё текст 

json - java script object notation

xml - легаси

Файлы передаются по HTTP, файлы передаются статические и динамические (ну понятно что одни могут меняться со временем а другие нет)

Если документ большой - то его разбивают на "чанки". Видео на стороне сервера лежит "целым"
И его декодят, разбивают видео на n файлов (картинки, аудиодорожки)
Веб сервер смотрит в директорию на диске и файл большой (клиент может отдавать файл по частям)
Клиент запрашивает участок файла и по чанкам выкачивает файл 

Путь к файлу - может быть динамический

опции запроса ака query string

Якорь - не попадает на сервер, когда получит документ, будет скролить документ до part1

Перед server могут добавит логин и пароль

десереализация - преобразование строки в джсон

curl -vv - verbose ответь мне всо


WEB-сервер - мощный сервер, который отдаёт статические документы

async defer - для бизнес анализа (чтобы не мешать прогрузке сайта)

загрузку CSS (link) - рекомендуется ставить в теге head, а script в конце, чтобы не стопить загрузку
